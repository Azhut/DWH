# Исправление логики построения иерархии в header_parsing.py

## Проблема

В функции `_estimate_depths_heuristic` в `app/domain/parsing/header_parsing.py` была некорректная логика обработки триггерных фраз ("из них", "в том числе"), что приводило к:

1. **Дубликатам ключей в MongoDB** - строки с одинаковыми координатами получали разные иерархические пути
2. **Некорректному flat_data_size** - из-за ошибок дубликатов при вставке
3. **Неправильной иерархии** - дочерние элементы не привязывались к правильным родителям

## Корень проблемы

```python
# Было (неправильно):
phrase_run_depth = max(base_level + 1, 1)  # всегда от base_level
```

Это приводило к тому, что "из них имеющие спортивные клубы" всегда получал глубину 1, независимо от контекста.

## Решение

### 1. Добавлен `current_context_depth`

```python
current_context_depth = 0  # Новая переменная - текущая глубина контекста
first_non_empty_processed = False  # Флаг первой непустой строки
```

### 2. Изменена логика обработки триггеров

```python
if opens_phrase:
    # Наращиваем глубину от текущего контекста, а не от base_level
    phrase_run_depth = current_context_depth + 1
    current_context_depth = phrase_run_depth  # Обновляем контекст
```

### 3. Правильная обработка подчиненных строк

```python
if dash:
    d = current_context_depth + 1  # Наращиваем от текущего контекста
    current_context_depth = d

if space_only_child:
    d = current_context_depth  # Сохраняем уровень контекста, не наращиваем
```

### 4. Корректный сброс контекста

```python
# Обновляем текущий контекст для базового уровня (без отступов)
if leading_spaces == 0 and not opens_phrase and not dash and not space_only_child:
    if not first_non_empty_processed:
        current_context_depth = base_level  # Первая строка - корень (depth=0)
        first_non_empty_processed = True
    else:
        current_context_depth = max(base_level, 1)  # Последующие строки - минимум depth=1
```

## Результат

### До исправления:
```
1. depth=0 | Всего учреждений...
2. depth=1 | в том числе: дошкольные...
3. depth=0 | общеобразовательные организации  # НЕПРАВИЛЬНО
4. depth=1 | из них имеющие спортивные клубы    # НЕПРАВИЛЬНО
```

### После исправления:
```
1. depth=0 | Всего учреждений...
2. depth=1 | в том числе: дошкольные...
3. depth=0 | общеобразовательные организации
4. depth=1 | из них имеющие спортивные клубы
5. depth=1 | организации профессионального образования
6. depth=2 | из них имеющие спортивные клубы  # ПРАВИЛЬНО - привязан к родителю
```

### Уникальные иерархические пути:
```
- Всего учреждений...
- общеобразовательные организации
- общеобразовательные организации | из них имеющие спортивные клубы
- организации профессионального образования
- организации профессионального образования | из них имеющие спортивные клубы
```
